# 개발의신 챗봇 사용 가이드

## 시작하기

### 개발 서버 실행

```bash
npm run dev
```

브라우저에서 `http://localhost:3000`을 열어 챗봇을 사용할 수 있습니다.

## 주요 기능

### 1. 대화 시작하기

- 처음 접속하면 자동으로 새로운 대화가 생성됩니다
- 하단 입력 필드에 질문을 입력하고 전송 버튼을 클릭하거나 Enter 키를 누릅니다
- Shift+Enter로 줄바꿈을 할 수 있습니다

### 2. 다중 대화 관리

#### 새 대화 생성
- 좌측 사이드바의 `+` 버튼을 클릭하여 새 대화를 시작합니다

#### 대화 전환
- 사이드바에서 원하는 대화를 클릭하여 전환할 수 있습니다
- 대화는 최근 업데이트 순으로 정렬됩니다

#### 대화 삭제
- 대화 항목에 마우스를 올리면 나타나는 휴지통 아이콘을 클릭합니다
- 확인 대화상자에서 삭제를 확인합니다

### 3. 모바일 사용

- 모바일에서는 좌측 상단의 메뉴 버튼(≡)을 눌러 사이드바를 열고 닫을 수 있습니다
- 대화를 선택하면 자동으로 사이드바가 닫힙니다

## Markdown 기능 활용

### 코드 블록

챗봇에게 코드 예시를 요청하면 문법 강조된 코드 블록으로 응답받을 수 있습니다:

```
"React에서 useState 사용 예시를 보여줘"
```

### 수식 표현

LaTeX 문법을 사용한 수학 공식을 표시할 수 있습니다:

```
"이차방정식의 해의 공식을 보여줘"
```

### 표와 리스트

구조화된 정보를 테이블이나 리스트 형태로 받을 수 있습니다:

```
"Python과 JavaScript의 차이점을 표로 정리해줘"
```

## 데이터 관리

### 로컬 저장소
- 모든 대화는 브라우저의 localStorage에 저장됩니다
- 서버나 클라우드에 데이터가 전송되지 않습니다
- 같은 브라우저에서 다시 접속하면 이전 대화를 계속할 수 있습니다

### 데이터 삭제
- 대화를 개별적으로 삭제하거나
- 브라우저의 저장소를 비워 모든 대화를 삭제할 수 있습니다

## 성능 최적화

### 응답 속도
- 스트리밍 방식으로 실시간 응답을 받습니다
- 네트워크 상태에 따라 속도가 달라질 수 있습니다

### 대화 히스토리
- 대화가 길어질수록 컨텍스트가 늘어나 응답 시간이 증가할 수 있습니다
- 새로운 주제로 대화할 때는 새 대화를 시작하는 것이 좋습니다

## 문제 해결

### 응답이 없을 때
1. 네트워크 연결 상태를 확인합니다
2. Gemini API 키가 올바르게 설정되었는지 확인합니다
3. 개발자 콘솔에서 에러 메시지를 확인합니다

### API 키 오류
```
Error: GEMINI_API_KEY is not set
```
- `.env.local` 파일에 올바른 API 키가 설정되어 있는지 확인합니다
- 서버를 재시작합니다

### 대화가 저장되지 않을 때
- 브라우저의 localStorage가 활성화되어 있는지 확인합니다
- 시크릿 모드에서는 localStorage가 제한될 수 있습니다

```typescript
    // ... 기타 색상
  },
}
```
